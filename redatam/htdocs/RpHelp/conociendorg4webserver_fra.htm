Présentation du R+G4 Web Server
[Pricipal] [Suivant]

   La quatrième génération du REDATAM (RG4) incorpore une nouvelle fonctionnalité visant le développement d’applications et destinée à faciliter le travail des usagers sur l’information utilisée dans des études et diagnostiques dont le contenu dépend des bases de données disponibles. Cette fonctionnalité permet de placer l’information dans un serveur de pages web pouvant recevoir des demandes de traitement des bases en format REDATAM, exécuter les instructions nécessaires et renvoyer le résultat demandé.

   A partir de sa version 1. 2  REDATAM permet d’accéder aux bases des données situées dans un serveur central, de telle manière que les usagers travaillant dans une institution donnée, peuvent installer le logiciel sur leurs machines et exécuter les traitements voulus. Ceci permet de libérer des ressources sur les machines locales et d’organiser les bases de données existantes dans un seul endroit. Cependant le traitement de chaque requête des usagers suppose un coût qui doit être administré par les machines locales et le serveur.

   Cette nouvelle fonctionnalité de REDATAM utilise la technologie et les langages développés dans le cadre  d’Intranet et Internet en utilisant des serveurs de pages web.  La différence avec le serveur central décrit précédemment réside dans le fait que l’application est exécutée para le serveur ce qui implique que les usagers n’ont pas besoin d’installer ni REDATAM ni les bases de données pour afficher les résultats des indicateurs ou croisements requis à un moment donné.
